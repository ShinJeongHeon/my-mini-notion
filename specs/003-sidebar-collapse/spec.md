# Feature Specification: 사이드바 접기/펼치기

**Feature Branch**: `003-sidebar-collapse`

**Created**: 2026-07-21

**Status**: Draft

**Input**: User description: "사이드 바에 접기/펼치기 기능을 구현해주세요. 사이드바에 버튼을 하나 만들고, 이걸 클릭하면 접기, 다시 클릭하면 펼치기가 되도록 해주세요."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 버튼 하나로 사이드바 접고 펼치기 (Priority: P1)

사용자가 사이드바에 있는 토글 버튼을 클릭하면 사이드바가 접혀 화면에서 사라지고,
본문(문서) 영역이 그만큼 넓어진다. 같은 버튼(펼치기 컨트롤)을 다시 클릭하면 사이드바가
원래 모습 그대로 다시 나타난다. 접힌 상태에서도 펼치기 컨트롤은 항상 화면에 남아 있어
사용자가 언제든 되돌릴 수 있다.

**Why this priority**: 사용자가 요청한 기능의 전부가 이 여정에 있다. 이 하나만 동작해도
"글쓰기에 집중할 넓은 화면"과 "탐색이 필요할 때 되돌리기"라는 핵심 가치를 완전히
전달한다.

**Independent Test**: 앱을 열고 토글 버튼을 클릭해 사이드바가 사라지는지, 다시 클릭해
사이드바가 이전과 동일한 내용으로 돌아오는지 확인하는 것만으로 독립적으로 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** 사이드바가 펼쳐진 기본 화면, **When** 사용자가 접기 버튼을 클릭하면,
   **Then** 사이드바 전체가 화면에서 숨겨지고 본문 영역이 사이드바가 차지하던
   폭만큼 넓어진다.
2. **Given** 사이드바가 접힌 화면, **When** 사용자가 펼치기 컨트롤을 클릭하면,
   **Then** 사이드바가 접기 전과 동일한 구성과 상태(검색어·활성 항목 등)로 다시
   표시된다. 접힌 사이 데이터가 변했다면 목록은 최신 상태를 반영한다.
3. **Given** 사이드바가 접힌 화면, **When** 사용자가 앱 내 어느 화면을 사용 중이든,
   **Then** 펼치기 컨트롤은 화면에 항상 보이며 클릭 가능한 상태로 남아 있다.
4. **Given** 사이드바 검색창에 검색어를 입력해 둔 상태, **When** 사이드바를 접었다가
   다시 펼치면, **Then** 입력해 둔 검색어와 필터링된 글 목록이 그대로 유지된다.

---

### User Story 2 - 페이지를 오가도 접힘 상태 유지 (Priority: P2)

사이드바를 접은 채 글 목록에서 다른 글로 이동하거나 홈·마이 페이지 등 앱 내 다른
화면으로 이동해도, 사이드바는 접힌 상태를 유지한다. 펼친 상태도 마찬가지로 유지된다.

**Why this priority**: 앱 내 이동 때마다 접힘 상태가 초기화되면 사용자는 화면을 옮길
때마다 버튼을 다시 눌러야 한다. P1의 가치를 실제 사용 흐름에서 지켜주는 요구사항이다.

**Independent Test**: 사이드바를 접은 뒤 앱 내 다른 화면으로 이동해 접힘 상태가
유지되는지 확인하는 것만으로 독립적으로 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** 사이드바를 접은 상태, **When** 본문 안의 링크나 이동 수단으로 앱 내 다른
   화면에 진입하면, **Then** 새 화면에서도 사이드바는 접힌 채 유지된다.
2. **Given** 사이드바를 펼친 상태, **When** 앱 내 다른 화면으로 이동하면, **Then** 새
   화면에서도 사이드바는 펼쳐진 채 유지된다.

---

### User Story 3 - 다시 방문해도 마지막 상태 기억 (Priority: P3)

사용자가 사이드바를 접어 둔 채 브라우저를 새로고침하거나 나중에 다시 방문하면,
사이드바는 마지막으로 선택한 접힘/펼침 상태로 나타난다.

**Why this priority**: 없어도 기능은 성립하지만, 매번 접기를 다시 눌러야 하는 반복을
없애 "내 작업 환경을 기억하는 앱"이라는 완성도를 더한다.

**Independent Test**: 사이드바를 접고 새로고침한 뒤 접힌 상태로 나타나는지 확인하는
것만으로 독립적으로 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** 사이드바를 접어 둔 상태, **When** 브라우저를 새로고침하면, **Then**
   사이드바는 접힌 상태로 나타난다.
2. **Given** 사이드바를 펼쳐 둔 상태(또는 저장된 선택이 없는 첫 방문), **When**
   앱에 진입하면, **Then** 사이드바는 펼쳐진 상태로 나타난다.

---

### Edge Cases

- 전환 애니메이션이 진행되는 중에 버튼을 연속으로 빠르게 클릭하면? → 클릭 횟수에 따른
  최종 토글 결과로 수렴해야 하며, 중간 상태에 멈추거나 깨진 레이아웃이 남으면 안 된다.
- 키보드만 사용하는 사용자가 토글할 수 있는가? → 버튼은 키보드 포커스와 Enter/Space로
  동작해야 하고, 접힌 동안 숨겨진 사이드바 내부 요소는 키보드 탐색 대상에서 제외되어야
  한다.
- 접힌 상태에서 새 글 작성 등으로 글 목록이 변해도, 다시 펼쳤을 때 목록은 최신 상태를
  반영해야 한다.
- 저장된 상태 값이 손상되었거나 읽을 수 없으면? → 기본값(펼침)으로 동작하고 오류를
  사용자에게 노출하지 않는다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 사이드바에는 접기/펼치기를 전환하는 토글 버튼이 하나 제공되어야 한다.
  클릭할 때마다 펼침 ↔ 접힘 상태가 번갈아 전환된다.
- **FR-002**: 접힘 상태에서는 사이드바의 전체 콘텐츠(예: 홈, 글 검색, 내 글 목록,
  앱 섹션, 프로필 영역 — 이후 사이드바에 추가되는 항목 포함)가 화면에서 숨겨지고,
  본문 영역이 확보된 폭을 사용해 넓어져야 한다.
- **FR-003**: 접힘 상태에서도 사이드바를 다시 펼칠 수 있는 컨트롤이 항상 화면에
  보이고 조작 가능해야 한다.
- **FR-004**: 토글 컨트롤은 마우스와 키보드 모두로 조작 가능해야 하며, 보조기술
  사용자가 현재 상태(접힘/펼침)와 동작을 인지할 수 있는 접근성 이름·상태를 제공해야
  한다.
- **FR-005**: 접힘/펼침 상태는 앱 내 화면 간 이동에서 유지되어야 한다.
- **FR-006**: 마지막으로 선택한 접힘/펼침 상태는 같은 기기·브라우저에서 새로고침 및
  재방문 시 복원되어야 한다. 저장된 값이 없거나 유효하지 않으면 펼침이 기본값이다.
- **FR-007**: 상태 전환은 화면 깜빡임이나 레이아웃 깨짐 없이 시작부터 완료까지 1초
  이내에 끝나야 하며, 전환 후 본문 콘텐츠는 즉시 상호작용 가능해야 한다.
- **FR-008**: 사이드바를 접었다 펼쳐도 사이드바 안의 모든 기능과 상태(예: 검색어,
  글 목록, 활성 항목 표시, 프로필 이동)가 손실되거나 초기화되어서는 안 된다.

### Key Entities

- **사이드바 표시 상태**: 사용자가 마지막으로 선택한 접힘/펼침 여부를 나타내는 단일
  선호 값. 사용자 기기에 저장되며 계정 간 동기화되지 않는다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자는 클릭(또는 키보드 조작) 한 번으로 사이드바를 접거나 펼 수 있으며,
  전환은 시작부터 완료까지 1초 이내에 끝난다.
- **SC-002**: 접힘 상태에서 본문 영역의 가용 폭은 사이드바가 차지하던 폭만큼 증가한다.
- **SC-003**: 접기 → 펼치기를 반복(10회 이상)해도 사이드바의 모든 기능이 100%
  동일하게 동작한다.
- **SC-004**: 앱 내 화면 이동과 새로고침 후에도 사용자가 마지막으로 선택한 상태가 100%
  유지된다(저장된 값이 유효한 경우 기준 — 손상·유실 시의 기본값 폴백은 FR-006과
  Edge Cases 참조).
- **SC-005**: 마우스 없이 키보드만으로 접기와 펼치기를 모두 완료할 수 있다.

## Assumptions

- 접힘은 사이드바 전체가 화면에서 숨겨지는 방식이다(아이콘만 남는 미니 레일 방식이
  아님). 이는 참조 제품(노션)의 관례를 따른 기본값이다.
- 접힌 상태에서 사용자가 다시 펼칠 수 있도록, 토글(펼치기) 컨트롤은 사이드바가 숨겨진
  뒤에도 접근 가능한 위치에 남는다 — "다시 클릭하면 펼치기"라는 요구의 전제 조건이다.
- 상태 저장은 사용자 기기(브라우저) 로컬 기준이며, 계정 단위 동기화는 범위 밖이다.
- 데스크톱 웹 화면을 기준으로 하며, 모바일 전용 반응형 동작(오버레이 등)은 이번 범위
  밖이다.
- 같은 시기에 진행되는 다크모드 기능(`003-dark-mode`)이 사이드바에 테마 토글 버튼을
  추가한다. 접힘 시 이 토글도 다른 콘텐츠와 함께 숨겨지며, 접힘 상태에서의 테마 전환
  수단은 다크모드 스펙에서 요구하지 않는다. 두 기능의 기기 로컬 선호값 저장은 동일한
  방식(저장·검증·폴백)을 공유해 구현할 수 있다.
