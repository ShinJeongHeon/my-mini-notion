# Feature Specification: 자기소개

**Feature Branch**: `003-profile-introduction`

**Created**: 2026-07-21

**Status**: Draft

**Input**: User description: "자기소개 기능을 구현해줘. 마이페이지에 현재 별명만 변경이 가능해. 추가로 자기소개도 등록/수정/조회 가능하도록 해줘. 내가 만든 DB테이블 구조는 변경하면 안돼."

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 자기소개 등록하기 (Priority: P1)

아직 자기소개를 등록하지 않은 사용자가 마이페이지에서 자기소개를 입력하고 저장한다.
마이페이지 프로필 카드에는 기존 별명 입력란 아래에 자기소개 입력란이 추가되고,
기존의 "변경사항 저장" 버튼 하나로 별명과 자기소개가 함께 저장된다.

**Why this priority**: 이 기능의 핵심 가치가 여기에 있다. 자기소개를 저장할 수 있어야
조회·수정도 의미가 생기므로, 이 하나만 동작해도 사용자는 요청한 목적(자기소개 등록)을
달성한다.

**Independent Test**: 자기소개가 비어 있는 계정으로 마이페이지에 진입해 자기소개를
입력하고 저장 버튼을 눌렀을 때 "저장되었습니다" 확인 표시가 나타나는 것만으로 독립적으로
검증 가능하다.

**Acceptance Scenarios**:

1. **Given** 자기소개가 등록되지 않은 사용자의 마이페이지, **When** 화면이 로드되면,
   **Then** 별명 입력란 아래에 비어 있는 자기소개 입력란과 입력을 유도하는 안내 문구
   (placeholder)가 표시된다.
2. **Given** 자기소개 입력란에 내용을 입력한 상태, **When** "변경사항 저장" 버튼을
   누르면, **Then** 자기소개가 저장되고 "저장되었습니다" 확인 표시가 잠시 나타난다.
3. **Given** 별명과 자기소개를 모두 수정한 상태, **When** "변경사항 저장" 버튼을 한 번
   누르면, **Then** 두 값이 모두 함께 저장된다.
4. **Given** 자기소개 입력 중 저장이 실패하는 상황(예: 네트워크 오류), **When** 저장을
   시도하면, **Then** 실패를 알리는 안내가 표시되고 입력한 내용은 사라지지 않고 유지된다.

---

### User Story 2 - 저장된 자기소개 조회하기 (Priority: P2)

자기소개를 등록해 둔 사용자가 마이페이지에 다시 들어왔을 때, 저장된 자기소개가
자기소개 입력란에 채워져 표시된다. 새로고침하거나 로그아웃 후 다시 로그인해도 동일하게
유지된다.

**Why this priority**: 저장(P1)이 되어도 다시 볼 수 없다면 등록의 가치가 없다. 조회는
등록 다음으로 중요한 기본 동작이다.

**Independent Test**: 자기소개를 저장한 뒤 페이지를 새로고침하고 마이페이지에 다시
진입해 저장했던 내용이 그대로 표시되는지 확인하는 것만으로 독립적으로 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** 자기소개를 저장한 사용자, **When** 마이페이지에 다시 진입하면, **Then**
   저장된 자기소개가 자기소개 입력란에 그대로 채워져 보인다.
2. **Given** 자기소개를 저장한 사용자, **When** 로그아웃 후 같은 계정으로 다시
   로그인해 마이페이지에 진입하면, **Then** 저장했던 자기소개가 그대로 표시된다.
3. **Given** 프로필 정보를 아직 불러오는 중인 마이페이지, **When** 로딩이 끝나면,
   **Then** 자기소개 입력란에 저장된 값이 채워진다(로딩 전 빈 값이 저장값을 덮어쓰지
   않는다).

---

### User Story 3 - 자기소개 수정·삭제하기 (Priority: P3)

이미 자기소개를 등록한 사용자가 마이페이지에서 내용을 고쳐 다시 저장하거나, 내용을
전부 지워 자기소개를 비운 상태로 저장한다.

**Why this priority**: 등록(P1)·조회(P2)가 갖춰진 뒤의 자연스러운 관리 동작이다. 등록과
같은 입력란·저장 버튼을 재사용하므로 별도 화면 없이 완성된다.

**Independent Test**: 기존 자기소개를 다른 내용으로 바꿔 저장한 뒤 새로고침해 변경된
내용이 표시되는지, 그리고 내용을 모두 지우고 저장한 뒤 빈 입력란과 안내 문구가
표시되는지 확인하는 것만으로 독립적으로 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** 자기소개가 등록된 사용자의 마이페이지, **When** 자기소개 내용을 수정하고
   "변경사항 저장"을 누르면, **Then** 수정된 내용이 저장되고 이후 조회 시 수정본이
   표시된다.
2. **Given** 자기소개가 등록된 사용자의 마이페이지, **When** 자기소개 내용을 모두
   지우고 저장하면, **Then** 자기소개가 등록되지 않은 상태로 돌아가며 이후 진입 시 빈
   입력란과 안내 문구가 표시된다.

---

### Edge Cases

- 최대 길이(500자)를 넘겨 입력하려 하면? — 입력이 500자에서 제한되어 그 이상 입력되지
  않는다(오류 안내가 아닌 입력 차단 방식).
- 줄바꿈이 포함된 여러 줄 자기소개를 저장하면? — 줄바꿈이 보존되어 다시 조회할 때
  동일한 줄 구성으로 표시된다.
- 별명만 바꾸고 자기소개는 그대로 저장하면? — 자기소개는 기존 값이 변하지 않고
  유지된다(그 반대의 경우도 동일).

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 마이페이지 프로필 카드에 자기소개 입력란이 표시되어야 한다. 자기소개가
  없으면 빈 입력란과 입력을 유도하는 안내 문구가 보여야 한다.
- **FR-002**: 사용자는 자기소개를 입력하고 기존 "변경사항 저장" 버튼으로 별명과 함께
  저장할 수 있어야 한다.
- **FR-003**: 저장된 자기소개는 마이페이지 재진입·새로고침·재로그인 후에도 유지되어
  자기소개 입력란에 표시되어야 한다.
- **FR-004**: 사용자는 저장된 자기소개를 수정하거나 전부 지워 비울 수 있어야 하며, 공백만
  있는 입력은 빈 자기소개로 처리되어야 한다.
- **FR-005**: 자기소개는 줄바꿈을 포함한 여러 줄 입력을 지원해야 하며, 최대 500자로
  제한되어야 한다.
- **FR-006**: 저장 실패 시 사용자에게 실패를 알리는 안내가 표시되어야 하고, 입력 중이던
  내용은 유실되지 않아야 한다.
- **FR-007**: 자기소개 기능은 사용자가 만든 기존 데이터 저장 구조(테이블·컬럼)를 변경·
  추가·삭제하지 않고, 현재 구조에 이미 존재하는 자기소개 저장 항목만 사용해 구현되어야
  한다.
- **FR-008**: 기존 별명 변경·프로필 이미지·이메일 표시·로그아웃 등 마이페이지의 다른
  기능은 기존과 동일하게 동작해야 한다.

### Key Entities

- **프로필**: 사용자당 하나 존재하는 개인 정보 묶음. 별명(name)과 자기소개(introduction)를
  담는다(선택 입력, 비어 있을 수 있음 — 항목 존재 확인은 Assumptions 참조). 본 기능은 이
  항목을 채우고 읽는 화면 동작만 추가한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자는 마이페이지 진입 후 1분 이내에 자기소개를 입력하고 저장까지 완료할
  수 있다.
- **SC-002**: 저장된 자기소개는 새로고침·재로그인을 거쳐도 100% 동일한 내용(줄바꿈 포함)
  으로 다시 표시된다.
- **SC-003**: 저장 실패 상황에서 사용자가 입력한 내용이 화면에서 유실되는 경우가 0건이다.
- **SC-004**: 기능 추가 이후에도 마이페이지의 기존 기능(별명 변경 저장 흐름 포함)이
  기능 추가 이전과 동일하게 동작한다(회귀 0건).

## Assumptions

- 자기소개의 등록·수정·조회 화면은 모두 마이페이지다. 다른 화면(사이드바, 게시글 등)에
  자기소개를 노출하는 것은 이번 범위에 포함하지 않는다.
- 자기소개는 본인만 조회·관리한다. 다른 사용자에게 공개하는 기능은 범위 밖이다.
- 자기소개 최대 길이는 500자로 한다(사용자 지정이 없어 통상적인 소개글 길이 기준을
  적용).
- 기존 데이터 저장 구조에는 자기소개를 담을 항목이 이미 존재함을 확인했으며, 따라서 저장
  구조 변경 없이 기능 구현이 가능하다.
