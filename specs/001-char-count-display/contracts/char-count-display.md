# UI Contract: 글자 수 표시 배지

이 프로젝트는 외부에 노출하는 API/CLI가 없는 클라이언트 웹 앱이므로, 여기서의 "계약"은
컴포넌트가 렌더링해야 하는 DOM 출력과 그 동작을 고정하는 것이다. 이 계약은
`/speckit-tasks`가 만들 테스트(RTL)와 `/speckit-implement`의 구현이 서로 어긋나지
않도록 하는 기준선이다.

## 위치 & 마크업

- **화면**: `app/(app)/posts/[id]/page.tsx` (게시글 상세/콘텐츠 편집 화면)
- **DOM 식별자**: `data-testid="content-char-count"`
- **표시 형식**: 숫자만(문자열) — 예: 빈 내용이면 정확히 `"0"`, 128자면 정확히 `"128"`.
  단위·라벨(`"자"` 등) 접두/접미사는 이 계약 버전에서 포함하지 않는다(spec FR-005,
  SC-002가 리터럴 `"0"` 표시를 요구).
- **위치 스타일**: `position: fixed`, 뷰포트(정확히는 `.app-main` 콘텐츠 영역) 기준
  우측 하단에 고정. 페이지를 스크롤해도 화면상 좌표가 바뀌지 않아야 한다
  (spec FR-003, Clarifications 2026-07-10).

## 입력 → 출력 계약

| 입력 (`post.content`) | 출력 (`data-testid="content-char-count"`의 텍스트) |
|---|---|
| `""` (빈 문자열) | `"0"` |
| `"안녕"` | `"2"` |
| `"a b"` (공백 포함 3글자) | `"3"` |
| `"line1\nline2"` (줄바꿈 포함) | `"11"` (줄바꿈 1글자 포함) |
| 사용자가 텍스트를 입력/삭제/붙여넣기 | 즉시(다음 렌더) 새 `content.length`로 갱신 |
| 다른 게시글로 이동 | 새로 열린 게시글의 `content.length`로 다시 계산되어 표시 |

## 비계약 사항 (범위 밖)

- 글자 수 상한, 경고 임계값, 색상 변화 등은 이 계약에 포함되지 않는다(spec Assumptions).
- 제목(title) 입력칸이나 목록 화면 미리보기는 이 계약의 대상이 아니다.
- 서버 API·영속화 계약 없음 — 순수 클라이언트 렌더링 계약이다.
