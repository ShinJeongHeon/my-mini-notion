## Clarifications

- Q: "우측 하단" 위치가 스크롤해도 항상 보이는 고정(floating) 위치인지, 본문과 함께
  스크롤되는 인라인 위치인지? → A: 편집 화면 우측 하단에 스크롤과 무관하게 항상 떠
  있는 고정(fixed/floating) 배지로 표시한다.

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 입력하면서 실시간으로 글자 수 확인하기 (Priority: P1)

글을 작성 중인 사용자가 내용 입력칸에 텍스트를 입력하거나 삭제하면, 화면 우측 하단에서
현재까지 입력한 글자 수를 실시간으로 확인할 수 있다.

**Why this priority**: 이 기능의 핵심 가치 전체가 여기에 있다. 이 하나만 동작해도
사용자는 원하는 목적(글자 수 파악)을 완전히 달성할 수 있다.

**Independent Test**: 게시글 상세 화면을 열고 내용 입력칸에 텍스트를 입력/삭제하면서
우측 하단의 숫자가 입력 내용과 함께 즉시 바뀌는지 확인하는 것만으로 독립적으로 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** 내용 입력칸이 비어 있는 게시글 상세 화면, **When** 사용자가 텍스트를 입력하면,
   **Then** 우측 하단에 입력한 글자 수와 동일한 숫자가 즉시 표시된다.
2. **Given** 내용 입력칸에 이미 텍스트가 있는 상태, **When** 사용자가 일부 텍스트를 삭제하면,
   **Then** 우측 하단의 글자 수가 줄어든 만큼 즉시 갱신된다.
3. **Given** 내용 입력칸에 텍스트를 붙여넣기(paste)한 경우, **When** 붙여넣기가 완료되면,
   **Then** 붙여넣은 전체 텍스트를 반영한 글자 수가 즉시 표시된다.

---

### User Story 2 - 빈 내용의 글자 수 확인 (Priority: P2)

내용 입력칸이 비어 있을 때도 사용자는 우측 하단에서 글자 수가 "0"으로 표시되는 것을
확인할 수 있어, 카운터가 정상 동작 중임을 알 수 있다.

**Why this priority**: 핵심 흐름은 아니지만, 카운터가 항상 화면에 존재하고 신뢰할 수
있다는 인상을 주는 데 필요하다.

**Independent Test**: 내용이 없는 새 게시글을 열었을 때 우측 하단에 "0"이 표시되는지
확인하는 것만으로 독립적으로 검증 가능하다.

**Acceptance Scenarios**:

1. **Given** 내용 입력칸이 완전히 비어 있는 상태, **When** 게시글 상세 화면을 열면,
   **Then** 우측 하단에 글자 수 "0"이 표시된다.

---

### Edge Cases

- 공백이나 줄바꿈만 입력했을 때도 각 문자가 글자 수에 포함되어 계산된다.
- 수천 자 이상의 매우 긴 내용을 입력해도 글자 수 표시가 지연되거나 부정확해지지 않는다.
- 다른 게시글로 이동했다가 돌아왔을 때, 글자 수는 새로 연 게시글의 실제 내용 기준으로
  다시 계산되어 표시된다.
- 텍스트를 빠르게 연속으로 입력하거나 삭제해도 화면에 표시되는 숫자가 입력 내용과
  어긋나지 않는다.
- 긴 내용을 작성 중 위/아래로 스크롤해도 글자 수 표시는 화면 우측 하단 같은 위치에
  계속 떠 있는 상태로 보인다.

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 시스템은 게시글 상세(콘텐츠 편집) 화면에서 내용 입력칸의 현재 글자 수를
  항상 화면에 표시해야 한다.
- **FR-002**: 시스템은 사용자가 내용 입력칸에 텍스트를 입력, 삭제, 붙여넣기할 때마다
  글자 수를 실시간으로 다시 계산하고 표시를 즉시 갱신해야 한다.
- **FR-003**: 글자 수 표시는 콘텐츠 편집 화면의 우측 하단에 스크롤 위치와 무관하게
  항상 같은 자리에 떠 있는(고정/floating) 형태로 위치해야 한다.
- **FR-004**: 시스템은 공백과 줄바꿈을 포함해 내용 입력칸에 존재하는 모든 문자를
  각각 하나의 글자로 계산해야 한다.
- **FR-005**: 내용 입력칸이 비어 있을 때 시스템은 글자 수를 "0"으로 표시해야 한다.
- **FR-006**: 사용자가 다른 게시글로 이동하면, 시스템은 새로 열린 게시글의 실제 내용을
  기준으로 글자 수를 다시 계산해 표시해야 한다.

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 사용자가 글자를 입력하거나 삭제한 시점부터 100ms 이내에 화면의 글자 수
  표시가 갱신된다.
- **SC-002**: 내용이 비어 있는 게시글을 열면 100%의 경우 글자 수가 "0"으로 표시된다.
- **SC-003**: 10,000자 이상의 긴 내용에서도 글자 수 표시가 지연 없이 정확하게 유지된다.
- **SC-004**: 별도 안내 없이도 사용자가 우측 하단의 숫자를 "현재까지 입력한 글자 수"로
  인지할 수 있다(사용성 확인 기준).

## Assumptions

- 글자 수는 단어 수가 아닌 문자(character) 단위로 계산하며, 공백과 줄바꿈도 포함한다.
- 이 기능은 게시글 상세 화면의 내용(content) 입력칸에만 적용되고, 제목 입력칸이나
  목록 화면의 미리보기에는 적용되지 않는다.
- 글자 수 표시는 별도의 저장이나 서버 전송 없이 클라이언트 화면에서만 계산·표시되는
  정보이며, 게시글 데이터 자체에 영향을 주지 않는다.
- 글자 수 상한이나 경고/제한 임계값에 대한 요구사항은 없다 — 현재 글자 수를 보여주는
  단순 표시 기능이다.
