# Quickstart: 콘텐츠 글자 수 표시 검증

이 기능이 실제로 동작하는지 확인하기 위한 최소 실행 경로다. 구현 세부 코드는
`tasks.md`(Phase 2)와 구현 단계에서 다루며, 여기서는 검증 시나리오만 다룬다.

## 사전 준비

```bash
npm install   # 아직 설치 전이라면
npm run dev   # http://localhost:3000
```

## 수동 검증 시나리오

1. **빈 글자 수 확인 (spec User Story 2 / SC-002)**
   - 브라우저에서 `/`로 이동 → 내용이 비어 있는 게시글을 열거나 새 페이지 생성.
   - 우측 하단에 글자 수 `0`이 표시되는지 확인.

2. **실시간 갱신 확인 (spec User Story 1 / SC-001)**
   - 내용 입력칸에 텍스트를 입력 → 우측 하단 숫자가 입력한 글자 수만큼 즉시 증가하는지 확인.
   - 일부 텍스트를 삭제 → 숫자가 즉시 감소하는지 확인.
   - 긴 텍스트를 붙여넣기(paste) → 붙여넣은 전체 길이가 반영되는지 확인.

3. **스크롤 시 고정 위치 확인 (Clarifications 2026-07-10 / FR-003)**
   - 내용 입력칸에 화면 높이보다 긴 텍스트를 입력.
   - 내용 영역을 위/아래로 스크롤 → 글자 수 배지가 화면 우측 하단 같은 자리에
     계속 떠 있는지(사라지거나 함께 스크롤되지 않는지) 확인.

4. **게시글 전환 시 재계산 확인 (spec FR-006)**
   - 다른 게시글로 이동(뒤로 가기 → 다른 글 클릭) → 새 글의 실제 내용 길이로
     글자 수가 다시 계산되어 표시되는지 확인.

## 자동화 검증

```bash
npm test                                   # 전체 테스트 (RED/GREEN 확인용)
npm test -- __tests__/post-detail-char-count.test.tsx   # 이 기능 테스트만
```

**기대 결과**: 위 테스트 파일이 [contracts/char-count-display.md](./contracts/char-count-display.md)의
입력→출력 표를 근거로 `data-testid="content-char-count"` 요소의 텍스트를 검증하며,
전체 스위트가 오류·경고 없이 통과해야 한다(헌법 완료 게이트).
